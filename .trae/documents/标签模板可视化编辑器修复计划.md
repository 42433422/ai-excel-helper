# 标签模板可视化编辑器修复计划

## 问题描述

用户反馈：
- 整个编辑器页面没有显示
- 之前左边出现的可以点击的展示（Canvas 预览）没有出现
- 之前下面选择框展示（属性面板）也没有了

## 已知信息

从之前的调试信息可知：
- `templateType=label` ✅ 正确
- `fields=6` ✅ 正确
- `grid=有` ✅ 正确

但 Canvas 预览和属性面板没有显示。

## 可能的问题原因

1. **布局问题**：
   - `.editor-layout` 的 CSS 可能被破坏
   - flex 布局可能没有正确应用

2. **组件渲染问题**：
   - LabelVisualEditor 组件可能没有正确挂载
   - 或者组件内部有错误导致渲染失败

3. **模板条件问题**：
   - `v-if="editorTemplateType === 'label'"` 条件可能有问题

## 修复步骤

### 1. 检查并修复布局结构

首先检查 LabelVisualEditor.vue 的 `.editor-layout` CSS：
```css
.editor-layout {
  display: flex;
  flex-direction: column;  /* 上下排列 */
  gap: 20px;
  height: 600px;
}
```

需要确保：
- 上下排列：Canvas 在上，属性面板在下
- Canvas 容器有足够的高度
- 属性面板有足够的高度

### 2. 简化组件结构进行调试

临时移除不必要的 div 嵌套，确保基本布局能显示：
```html
<div class="visual-editor">
  <div class="editor-layout">
    <!-- 标签预览 -->
    <div class="preview-section">
      <canvas ref="labelCanvas" width="900" height="600"></canvas>
    </div>

    <!-- 属性面板 -->
    <div class="properties-section">
      属性面板内容
    </div>
  </div>
</div>
```

### 3. 检查 Canvas 和属性面板是否都渲染

添加调试信息：
```html
<div class="preview-section">
  <h4>标签预览</h4>
  <div class="canvas-container">
    Canvas 容器
    <canvas ref="labelCanvas"></canvas>
  </div>
</div>

<div class="properties-section" v-if="selectedField">
  属性面板
</div>
<div class="properties-section empty-state" v-else>
  点击字段进行编辑
</div>
```

### 4. 确保数据正确传递

检查 props：
- `fields`: 数组
- `grid`: 对象
- `imageSize`: {width, height}

### 5. 验证渲染流程

1. mounted 钩子应该执行
2. watch 监听 fields 变化并调用 drawCanvas
3. drawCanvas 应该绘制背景、网格、字段
