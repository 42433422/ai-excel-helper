# XCAGI 项目架构改进执行计划

## 📋 计划概述

基于全面架构审查，本计划针对识别出的关键问题提供系统性改进方案，按优先级分阶段执行。

---

## Phase 1：数据库层解耦 (高优先级)

### 目标
消除 `services/` 层对 SQLAlchemy 的直接依赖，建立正确的仓储模式。

### 具体步骤

#### 1.1 创建 Materials 仓储接口
- 创建 `app/application/ports/material_repository.py` 定义 `MaterialRepository` 接口
- 定义标准 CRUD 方法签名

#### 1.2 实现 SQLAlchemy 仓储
- 创建 `app/infrastructure/persistence/material_repository_impl.py`
- 将 `MaterialsService` 中的所有 `get_db()` 操作迁移至此
- 保持原有的查询逻辑和分页能力

#### 1.3 改造 MaterialsService
- 重构 `MaterialsService`，移除所有 `get_db()` 调用
- 通过构造函数注入 `MaterialRepository`
- 保留业务逻辑（验证、转换）

#### 1.4 更新 MaterialApplicationService
- 接收 `MaterialRepository` 而非直接操作数据库
- 在应用层管理事务边界

#### 1.5 验证测试
- 确保 `GET /api/materials` 正常工作
- 确保 `POST /api/materials` 正常工作
- 运行现有单元测试

---

## Phase 3：意图识别系统重构 (中优先级)

### 目标
将 595 行的巨型 `intent_service.py` 拆分为单一职责模块。

### 具体步骤

#### 3.1 创建策略类
- `app/domain/services/intent/greeting_strategy.py` - 问候语识别
- `app/domain/services/intent/goodbye_strategy.py` - 告别语识别
- `app/domain/services/intent/tool_intent_strategy.py` - 工具意图识别

#### 3.2 提取规则引擎
- 将 `rule_engine.py` 移至 `app/infrastructure/rules/rule_engine.py`
- 创建 `IntentRuleRegistry` 统一管理规则配置

#### 3.3 重构 IntentService
- 作为组合层，协调各策略
- 保持 `recognize_intents()` 和 `quick_recognize()` 接口不变

## Phase 4：技术债务清理 (低优先级)

### 目标
现代化框架和工具使用

### 具体步骤

#### 4.1 引入依赖注入容器
- 评估 `punq` 或 `dependency-injector`
- 替换 `bootstrap.py` 中的手写工厂函数

#### 4.2 Redis 缓存充分利用
- 检查 `config.py` 中的 Redis 配置
- 在 `session.py` 中启用 Redis 缓存

#### 4.3 测试覆盖率提升
- 为核心领域模型添加单元测试
- 建立 CI/CD 测试门禁
