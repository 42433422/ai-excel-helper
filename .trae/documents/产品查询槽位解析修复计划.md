# 产品查询槽位解析修复计划

## 问题描述
用户输入 "帮我查询蔺宇的 2737A 是什么" 时：
- 意图识别正确：`products`
- 但槽位只有 `{'keyword': '蔺宇的 2737A'}`
- 没有解析出 `unit_name` 和 `model_number`
- 导致产品查询无法正确执行

## 根本原因
`deepseek_intent_service.py` 中的 `_extract_slots_from_rule` 方法：
1. 可能 keyword 解析逻辑没有被执行
2. 或者正则表达式匹配失败
3. 或者拼音模糊匹配失败

## 修复步骤

### 1. 检查当前槽位解析逻辑
- 查看 `_extract_slots_from_rule` 方法
- 验证 keyword 解析正则表达式
- 检查 resolve_purchase_unit 是否被调用

### 2. 添加调试日志
在 `_extract_slots_from_rule` 中添加：
```python
logger.info(f"[SLOT_EXTRACTION] keyword={keyword}, unit_match={unit_match}")
logger.info(f"[SLOT_EXTRACTION] resolved={resolved}")
```

### 3. 修复槽位解析
确保以下逻辑正确执行：
```python
if "keyword" in slots and "unit_name" not in slots:
    keyword = slots["keyword"]
    unit_match = re.search(r'([^\s 的]{2,6}) 的 (\d{4}[A-Z]?)', keyword)
    if unit_match:
        potential_unit = unit_match.group(1)  # 蔺宇
        model = unit_match.group(2)  # 2737A
        resolved = resolve_purchase_unit(potential_unit)
        if resolved:
            slots["unit_name"] = resolved.unit_name  # 澜宇
            slots["model_number"] = model
```

### 4. 验证修复
测试用例：
- "帮我查询蔺宇的 2737A 是什么"
- 期望 slots: `{'keyword': '蔺宇的 2737A', 'unit_name': '澜宇', 'model_number': '2737A'}`

### 5. 清理调试日志
修复完成后移除调试日志

## 验收标准
1. 用户输入 "帮我查询蔺宇的 2737A 是什么"
2. 后端日志显示：
   - `unit_name=澜宇`
   - `model_number=2737A`
3. 前端自动打开产品列表
4. 前端自动选择"澜宇"单位
5. 前端自动过滤到"2737A"型号
