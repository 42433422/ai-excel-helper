# 标签模板可视化编辑重构计划

## 需求分析

### 当前问题
1. 字段列表是纯文本输入框，不够直观
2. 无法看到标签的实际布局
3. 字段位置需要可视化调整
4. 固定词条和可变词条的区分不够清晰

### 目标效果
创建一个**悬浮式可视化编辑器**，类似设计软件：
- 显示标签的预览图（类似原图片布局）
- 点击字段即可编辑
- 字段用不同颜色区分类型（固定/可变）
- 可以拖拽调整字段位置
- 原始数值用 "X" 占位符显示

## 设计方案

  confidence: 0.9996

### 3. 交互设计

#### 3.1 标签预览区
- 使用 Canvas 绘制标签背景（白色矩形 + 黑色边框）
- 根据识别的网格线绘制表格
- 每个字段渲染为一个**可点击的矩形框**
  - 固定字段：蓝色边框
  - 可变字段：绿色边框
  - 悬停时：高亮显示 + 显示编辑图标

#### 3.2 字段编辑
- **点击字段** → 右侧属性面板显示该字段信息
- **拖拽字段** → 可调整位置（可选功能）
- **双击字段** → 直接编辑文本

#### 3.3 属性面板
```

```

### 4. 技术实现

#### 4.1 新增组件
`LabelVisualEditor.vue`
- Props:
  - `fields`: 识别的字段数组
  - `grid`: 网格配置（可选）
  - `imageSize`: 图片尺寸
- Events:
  - `field-change`: 字段变化
  - `fields-update`: 批量更新字段

#### 4.2 Canvas 渲染逻辑
```javascript
// 绘制标签背景
ctx.fillStyle = '#ffffff'
ctx.fillRect(0, 0, width, height)
ctx.strokeStyle = '#000000'
ctx.lineWidth = 2
ctx.strokeRect(0, 0, width, height)

// 绘制网格线
horizontalLines.forEach(y => {
  ctx.beginPath()
  ctx.moveTo(0, y)
  ctx.lineTo(width, y)
  ctx.stroke()
})

verticalLines.forEach(x => {
  ctx.beginPath()
  ctx.moveTo(x, 0)
  ctx.lineTo(x, height)
  ctx.stroke()
})

// 绘制字段
fields.forEach(field => {
  const isSelected = field.id === selectedFieldId
  ctx.fillStyle = field.type === 'fixed' ? '#e3f2fd' : '#e8f5e9'
  ctx.strokeStyle = isSelected ? '#ff9800' : (field.type === 'fixed' ? '#2196f3' : '#4caf50')
  ctx.lineWidth = isSelected ? 3 : 1
  
  // 绘制字段背景
  ctx.fillRect(field.position.x, field.position.y, field.position.width, field.position.height)
  ctx.strokeRect(field.position.x, field.position.y, field.position.width, field.position.height)
  
  // 绘制字段文本（标签：值）
  ctx.fillStyle = '#000000'
  ctx.font = '16px Arial'
  ctx.fillText(`${field.label}: ${field.value || 'X'}`, field.position.x + 10, field.position.y + 30)
})
```

#### 4.3 交互实现
```javascript
// Canvas 点击检测
canvas.addEventListener('click', (e) => {
  const rect = canvas.getBoundingClientRect()
  const x = (e.clientX - rect.left) * (canvas.width / rect.width)
  const y = (e.clientY - rect.top) * (canvas.height / rect.height)
  
  // 检测点击了哪个字段
  const clickedField = fields.find(field => 
    x >= field.position.x && 
    x <= field.position.x + field.position.width &&
    y >= field.position.y && 
    y <= field.position.y + field.position.height
  )
  
  if (clickedField) {
    selectedFieldId.value = clickedField.id
    emit('field-selected', clickedField)
  }
})

// 拖拽实现（可选）
let isDragging = false
let dragOffset = { x: 0, y: 0 }

canvas.addEventListener('mousedown', (e) => {
  const field = getFieldAtPosition(e)
  if (field) {
    isDragging = true
    dragOffset.x = e.offsetX - field.position.x
    dragOffset.y = e.offsetY - field.position.y
  }
})

canvas.addEventListener('mousemove', (e) => {
  if (isDragging && selectedField) {
    selectedField.position.x = e.offsetX - dragOffset.x
    selectedField.position.y = e.offsetY - dragOffset.y
    redrawCanvas()
  }
})

canvas.addEventListener('mouseup', () => {
  isDragging = false
})
```

#### 保存模板
```javascript
async saveTemplate() {
  const fields = this.$refs.visualEditor.getFields()
  
  // 构建保存数据
  const templateData = {
    name: this.templateName,
    type: this.editorTemplateType,
    fields: fields.map(f => ({
      label: f.label,
      value: f.value,
      type: f.type,
      position: f.position  // 包含位置信息
    }))
  }
  
  // 调用保存 API
  await saveTemplate(templateData)
}
```

## 预期效果

### 识别后
用户看到：
- 一个和原图布局一致的标签预览
- 每个字段用彩色框标出
- 字段内显示 "标签：X"（X 代表可变值）

### 编辑时
用户可以：
- 点击字段 → 右侧显示属性面板
- 修改字段名、字段值
- 切换固定/可变类型
- 拖拽调整位置（可选）

### 保存后
数据结构包含：
- 字段名
- 字段值（示例值）
- 字段类型（固定/可变）
- **字段位置**（x, y, width, height）

## 技术要点

1. **Canvas 缩放**：确保在不同屏幕上显示一致
2. **坐标转换**：Canvas 坐标 ↔ 实际图片坐标
3. **性能优化**：避免频繁重绘 Canvas
4. **响应式**：支持不同尺寸的标签图片
