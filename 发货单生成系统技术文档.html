发货单生成系统 - 技术文档

1. 系统概述

发货单生成系统是一个完整的解决方案，用于从用户输入的订单信息自动生成发货单Excel文档和商标标签。系统支持智能解析自然语言订单文本，处理标准和非标准产品，并通过OCR技术验证标签内容的正确性。

1.2 核心功能

  • 智能订单解析：从自然语言文本中提取结构化订单信息
  • 自定义模式：处理非标准产品，保留用户输入的产品信息
  • Excel文档生成：使用模板生成标准发货单文档
  • 商标标签生成：通过BarTender软件生成标准化标签
  • OCR标签验证：使用OCR技术验证标签内容的正确性
  • 数据库整合：从SQLite数据库中获取产品和客户信息
  • Web界面：提供直观的用户界面，支持订单输入和结果下载

3. 核心组件

ShipmentParser
AI助手/shipment_parser.py
智能订单解析器，负责从自然语言文本中提取结构化订单信息

主要功能：

  • 提取购买单位信息
  • 分割多个产品条目
  • 解析产品名称、型号、数量、规格等信息
  • 支持自定义模式处理非标准产品
  • 从数据库获取产品和客户信息
DocumentAPIGenerator
AI助手/shipment_document.py
文档生成器，使用模板生成Excel发货单文档

主要功能：

  • 加载Excel模板文件
  • 填充订单数据到模板中
  • 处理多产品订单
  • 保存生成的Excel文档
  • 返回文档下载链接
LabelGenerator
AI助手/BarTender_Template_Tools/label_generator.py
标签生成器，从订单数据生成商标标签信息

主要功能：

  • 从订单数据中提取标签所需信息
  • 准备标签数据结构
  • 格式化数据以符合标签模板要求
  • 调用BarTenderIntegrator生成标签
BarTenderIntegrator
AI助手/BarTender_Template_Tools/bartender_integrator.py
BarTender集成器，连接BarTender软件生成标签

主要功能：

  • 连接到BarTender应用程序
  • 打开BarTender模板文件
  • 获取模板信息和子字符串
  • 填充标签数据
  • 生成标签文件
OCRTextProcessor
AI助手/BarTender_Template_Tools/bartender_ocr_backend.py
OCR文字处理器，识别标签内容并验证正确性

主要功能：

  • 截取BarTender标签窗口
  • 使用EasyOCR识别标签上的文字
  • 确定文字在标签上的位置
  • 验证标签信息的正确性
  • 修改标签上的文字内容

3. API接口

3.1 生成发货单

POST
/api/generate

请求参数：

参数名 类型 必填 描述
order_text string 是 订单文本信息
template_name string 否 模板文件名，默认"尹玉华1.xlsx"

5. 技术实现细节

5.1 智能订单解析

订单解析器使用正则表达式和规则引擎从自然语言文本中提取结构化信息。主要步骤包括：

  • 提取购买单位：通过匹配已知单位列表和数据库中的单位信息
  • 分割产品：按逗号分割多个产品条目
  • 解析产品信息：使用正则表达式提取产品名称、型号、数量、规格等
  • 自定义模式处理：当启用自定义模式时，保留用户输入的产品信息，不强制匹配数据库

5.2 文档生成

文档生成器使用OpenPyXL库加载Excel模板，填充订单数据，然后保存为新的Excel文件。主要步骤包括：

  • 加载模板文件：使用指定的模板文件名
  • 定位数据区域：在模板中找到需要填充数据的单元格
  • 填充订单数据：购买单位、产品列表、数量、金额等
  • 处理多产品：如果订单包含多个产品，自动扩展表格
  • 保存文件：将生成的文档保存到outputs目录

5.3 标签生成

标签生成通过BarTenderIntegrator组件与BarTender软件交互，实现自动化标签生成。主要步骤包括：

  • 连接BarTender：使用COM接口创建BarTender应用实例
  • 打开模板：加载指定的BarTender模板文件
  • 填充数据：将标签数据填充到模板的相应字段
  • 生成标签：使用BarTender的API生成标签文件
  • 保存结果：将生成的标签保存为PDF或其他格式

5.4 OCR验证

OCR验证使用EasyOCR库识别标签上的文字，验证标签信息的正确性。主要步骤包括：

  • 窗口截图：截取BarTender标签窗口的内容
  • 文字识别：使用EasyOCR识别标签上的文字及其位置
  • 内容验证：检查识别的文字是否与预期内容一致
  • 位置验证：确保文字在标签上的位置正确
  • 结果保存：将验证结果保存到日志文件

6. 部署指南

6.1 环境要求

  • Windows操作系统
  • Python 3.12或更高版本
  • BarTender软件（用于标签生成）
  • 所需Python库：Flask、OpenPyXL、EasyOCR、win32com、SQLite3

6.2 安装步骤

  1. 安装Python：下载并安装Python 3.12或更高版本
  2. 安装依赖：在项目目录中运行 pip install -r requirements.txt
  3. 安装BarTender：下载并安装BarTender软件
  4. 配置环境：确保BarTender已正确安装并可以通过COM接口访问
  5. 启动服务：运行 python app_api.py 或使用启动器批处理文件

6.3 启动方式

系统提供了两种启动方式：

  1. 命令行启动：在项目目录中运行 python app_api.py
  2. 批处理文件启动：双击运行 发货单启动器2.0.bat 文件

6.4 配置说明

  • 模板文件：将Excel模板文件放在项目根目录中
  • BarTender模板：将BTW模板文件放在项目根目录中
  • 数据库文件：系统使用SQLite数据库，数据库文件为 products.db
  • 输出目录：生成的文档和标签保存在 outputs 目录中

7. 故障排除

7.1 常见问题

7.1.1 BarTender连接失败

  • 症状：标签生成失败，错误信息显示无法连接到BarTender
  • 原因：BarTender未安装或未正确配置
  • 解决方法：
    • 确保BarTender软件已正确安装
    • 检查BarTender是否已启动
    • 确保当前用户有足够的权限访问BarTender
    • 检查COM接口是否正常工作

7.1.2 OCR识别失败

  • 症状：标签验证失败，无法识别标签上的文字
  • 原因：EasyOCR未正确安装或标签图像质量差
  • 解决方法：
    • 确保EasyOCR库已正确安装
    • 检查标签图像是否清晰
    • 调整OCR识别参数

7.1.3 订单解析失败

  • 症状：无法从订单文本中提取有效信息
  • 原因：订单文本格式不正确或产品信息不存在于数据库中
  • 解决方法：
    • 检查订单文本格式是否正确
    • 启用自定义模式处理非标准产品
    • 确保数据库中存在相关产品信息

8. 安全注意事项

8.1 安全考虑

  • 文件上传：系统不接受文件上传，只处理用户输入的文本信息
  • 路径遍历：在文件下载API中实现了路径遍历防护，确保用户只能下载outputs目录中的文件
  • SQL注入：使用参数化查询，防止SQL注入攻击
  • 输入验证：对用户输入进行验证，确保输入符合预期格式
  • 错误处理：实现了完整的错误处理，避免敏感信息泄露

8.2 最佳实践

  • 定期备份数据库文件
  • 限制系统的访问权限
  • 定期更新依赖库
  • 监控系统日志，及时发现异常
