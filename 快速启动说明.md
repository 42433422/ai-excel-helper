# XCAGI v4.0 AI 员工 Docker 快速启动指南

## ⚡ 三步快速启动

### 方式 1：使用部署脚本（最简单）

1. **双击运行** `deploy.bat`
2. **选择选项** `1` (生产环境部署)
3. **等待完成**，看到访问地址

### 方式 2：使用快速启动脚本

```cmd
双击 quick-start.bat
```

### 方式 3：手动命令行

```bash
# 1. 进入项目目录
cd E:\FHD\XCAGI

# 2. 创建环境变量文件（首次运行）
copy .env.example .env

# 3. 配置 AI 员工模式（可选）
# 编辑 .env 文件，添加：
# AI_EMPLOYEE_MODE=true
# INDUSTRY_TYPE=manufacturing

# 4. 构建并启动
docker-compose up -d --build

# 5. 查看状态
docker-compose ps

# 6. 查看日志
docker-compose logs -f
```

## 🤖 AI 员工配置

### 配置 AI 员工模式

编辑 `.env` 文件：

```bash
# AI 员工模式
AI_EMPLOYEE_MODE=true

# 行业类型 (manufacturing/retail/logistics/wholesale/ecommerce)
INDUSTRY_TYPE=manufacturing

# TTS 语音设置
TTS_ENABLED=true
TTS_VOICE=zh-CN-XiaoxiaoNeural

# 自动化设置
AUTO_PROCESS_SHIPMENT=true
AUTO_PROCESS_RECEIVE=true
```

## 🔧 解决"no configuration file provided"错误

### 问题原因
运行 `docker-compose` 命令时不在项目目录下。

### 解决方法

**方法 1：先切换目录**
```cmd
cd E:\FHD\XCAGI
docker-compose logs -f
```

**方法 2：使用完整路径**
```cmd
docker-compose -f E:\FHD\XCAGI\docker-compose.yml logs -f
```

**方法 3：使用部署脚本**（推荐）
```cmd
deploy.bat
```
脚本会自动切换到正确目录。

## 📋 常用命令速查

### 启动服务
```bash
# 生产环境
docker-compose up -d --build

# 开发环境
docker-compose -f docker-compose.dev.yml up -d --build
```

### 查看状态
```bash
# 查看所有容器
docker-compose ps

# 查看详细信息
docker-compose top
```

### 查看日志
```bash
# 所有服务日志
docker-compose logs -f

# 特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f redis
docker-compose logs -f celery-worker
```

### 停止服务
```bash
# 停止所有
docker-compose down

# 停止并清理数据（危险！）
docker-compose down -v
```

### 重启服务
```bash
# 重启所有
docker-compose restart

# 重启单个服务
docker-compose restart backend
```

### 进入容器
```bash
# 进入后端容器
docker exec -it xcagi-backend bash

# 进入 Redis 容器
docker exec -it xcagi-redis redis-cli
```

### AI 员工状态检查
```bash
# 查看 AI 员工运行状态
docker exec xcagi-backend python -c "from app.services.system_service import get_system_info; print(get_system_info())"

# 查看意图识别引擎状态
docker exec xcagi-backend python -c "from app.services.unified_intent_recognizer import UnifiedIntentRecognizer; print(UnifiedIntentRecognizer().get_status())"
```

## ❓ 常见问题

### Q1: "no configuration file provided"
**解决**：确保在 `E:\FHD\XCAGI` 目录下运行命令

### Q2: 端口被占用
**解决**：
```bash
# 查看占用端口的进程
netstat -ano | findstr :5000
# 或修改 docker-compose.yml 中的端口映射
```

### Q3: 服务启动后立即停止
**解决**：
```bash
# 查看详细日志
docker-compose logs backend
# 检查 .env 配置
```

### Q4: 前端无法访问
**解决**：
```bash
# 检查前端容器状态
docker-compose ps frontend
# 重新构建前端
docker-compose build frontend
docker-compose up -d frontend
```

### Q5: AI 员工无法识别意图
**解决**：
```bash
# 检查 BERT 模型是否加载
docker-compose logs backend | grep "BERT"
# 重新加载意图引擎
curl -X POST http://localhost:5000/api/ai/reload-intent
```

### Q6: TTS 语音不工作
**解决**：
```bash
# 检查 TTS 配置
docker-compose logs backend | grep "TTS"
# 确保 .env 中设置了 TTS_ENABLED=true
```

## 🔄 升级到 V4.0

如果从 v3.0 升级：

```bash
# 1. 备份数据
docker-compose down
docker run --rm -v xcagi-data:/data -v $(pwd)/backup:/backup alpine tar czf /backup/xcagi-data-backup.tar.gz /data

# 2. 更新代码
git pull origin main

# 3. 更新 .env 配置
# 添加 AI_EMPLOYEE_MODE=true

# 4. 重新构建
docker-compose build --no-cache

# 5. 启动服务
docker-compose up -d

# 6. 运行数据库迁移
docker exec xcagi-backend alembic upgrade head
```
