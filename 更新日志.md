# XCAGI Docker 部署更新日志

## [2026-03-25] - Docker Compose 配置优化

### ✨ 新增功能

1. **一键部署脚本**
   - `deploy.bat` - 交互式部署工具（8 个选项）

   - `logs.bat` - 实时日志查看脚本
   - `health-check.bat` - 健康检查脚本

2. **文档**

   - `快速启动说明.md` - 快速入门文档

1. **Docker Compose 配置更新**
   - 移除 `docker-compose.yml` 中的 `version` 字段（已废弃）
   - 移除 `docker-compose.dev.yml` 中的 `version` 字段
   - 消除 Docker Compose 警告信息

2. **部署脚本增强**
   - 添加配置文件检查
   - 自动切换工作目录
   - 更友好的错误提示

### 📋 使用说明

#### Windows 用户

**最简单方式**：
```cmd
双击 quick-start.bat
```

**交互式部署**：
```cmd
双击 deploy.bat
选择选项：1 (生产环境部署)
```

**查看日志**：
```cmd
双击 logs.bat
```

**健康检查**：
```cmd
双击 health-check.bat
```

#### Linux/Mac 用户

```bash
# 赋予执行权限
chmod +x deploy.sh

# 运行部署
./deploy.sh
```

### 🔍 常见问题解决

#### 1. "no configuration file provided"

**原因**：不在项目目录运行命令

**解决**：
```cmd
cd E:\FHD\XCAGI
docker-compose up -d
```

或使用部署脚本（推荐）：
```cmd
双击 deploy.bat
```

#### 2. 端口被占用

**解决**：
```bash
# 查看占用进程
netstat -ano | findstr :5000

# 或修改 docker-compose.yml 端口映射
```

#### 3. 服务启动失败

**解决**：
```bash
# 查看详细日志
docker-compose logs backend

# 检查 .env 配置
```

### 🛠️ 技术细节

#### Docker Compose 版本说明

- **旧版本**：需要 `version: "3.8"` 字段
- **新版本**：`version` 字段已废弃，会自动忽略
- **建议**：移除该字段以避免警告

#### 多阶段构建

后端 Dockerfile 使用多阶段构建：
1. **Builder** - 编译依赖
2. **Production** - 最小化运行环境
3. **Development** - 开发调试环境

#### 数据持久化

使用 Docker 卷存储重要数据：
- `xcagi-data` - 数据库文件
- `xcagi-uploads` - 上传文件
- `xcagi-logs` - 日志文件
- `redis-data` - Redis 数据
