# 普通版发货单处理逻辑完整路线

> 文档版本：v1.0  
> 最后更新：2026-03-26  
> 适用范围：XCAGI 发货单生成系统（普通版 - 分支 A）

---

## 📋 总览

普通版发货单处理流程分为**五个阶段**：

1. **意图识别** → 任务创建
2. **订单解析** → 预览生成
3. **文档生成** → 数据持久化
4. **返回结果** → 前端展示
5. **标签生成** → 打印机对接

**处理流程**:

### 2.3 订单文本解析核心逻辑

**文件**: `app/routes/tools.py` - `_parse_order_text()`

```python
def _parse_order_text(order_text: str) -> dict:
    """
    解析订单文本，提取单位名和产品信息
    
    支持格式:
    - "发货单七彩乐园 1 桶 9803 规格 12"
    - "送货单公司名称 2 箱产品 A 规格 100"
    - "给我打印七彩乐园发货单，编号 9803，规格二十八，一共三桶"
    """
```

**解析步骤**:

```
1. 文本预处理
   ├─ 移除"发货单/送货单/出货单"关键词
   ├─ 替换中文标点为空格
   └─ 归一化"的规格" → "规格"

2. 槽位提取（支持任意语序）
   │
   ├─ 提取单位名 (_cleanup_unit_name)
   │   ├─ 移除命令词："打印"、"帮我"、"给我"
   │   ├─ 移除增删改词："再加"、"删除"、"改成"
   │   ├─ 移除型号数字串
   │   └─ 清理空格和标点
   │
   ├─ 提取型号 (model_number)
   │   ├─ 匹配"编号 XXX"或"型号 XXX"
   │   └─ 兜底：匹配"规格"前的数字串
   │
   ├─ 提取规格 (tin_spec)
   │   ├─ 支持阿拉伯数字：28
   │   ├─ 支持中文数字：二十八
   │   └─ 支持连读："规格 28 要 3 桶"
   │
   └─ 提取桶数 (quantity_tins)
       ├─ 匹配"X 桶"
       └─ 支持中文数字：三桶

3. 产品构建
   └─ 生成 products 数组

4. 返回解析结果
   {
     "success": True/False,
     "unit_name": "单位名称",
     "products": [{
       "name": "产品名称",
       "quantity_tins": 桶数，
       "tin_spec": 每桶规格，
       "model_number": "型号"
     }],
     "message": "错误消息（如果有）"
   }
```

**中文数字支持**:

```python
def _parse_cn_number(token: str):
    """解析阿拉伯数字/常见中文数字"""
    # 支持：零〇一二两三四五六七八九十二十八三十
    # 示例：
    #   "二十八" → 28
    #   "三十" → 30
    #   "三桶" → 3
```

**打印前询问（新增约束）**:

```json
{
  "require_confirm": true,
  "confirm_token": "print_confirm_xxx",
  "confirm_prompt": "已生成 3 张标签，是否立即打印到【TSC TTP-244】？",
  "preview": {
    "order_number": "26-0300123A",
    "label_count": 3,
    "printer": "TSC TTP-244"
  }

```

**前端交互规范**:

1. 收到 `require_confirm=true` 时，不直接调用打印接口。  
2. 先弹窗询问用户：
   - 文案：`是否立即打印？`
   - 按钮：`确认打印` / `取消`
3. 用户点击：
   - `确认打印` → 调用 `POST /api/print/label`，并透传 `confirm_token`
   - `取消` → 不打印，返回 `"print_cancelled"` 状态给任务卡

**打印流程**:

```python
PrinterService.print_label(file_path, printer_name, copies)
    │
    ├─ 0. 校验确认令牌（confirm_token）
    │   ├─ 缺少令牌且 require_confirm=true → 拒绝打印
    │   └─ 令牌无效/过期 → 返回 "print_confirm_required"
    │
    ├─ 1. 确定标签打印机
    │   ├─ 如果指定了 printer_name → 使用指定的
    │   └─ 否则 → get_label_printer() 自动获取
    │
    ├─ 2. 验证打印机可用性
    │   └─ 如果未找到 → 返回错误："未找到标签打印机"
    │
    ├─ 3. 循环打印（支持多份）
    │   └─ for i in range(copies):
    │       └─ printer_utils.print_file(file_path, printer_name)
    │
    ├─ 4. 统计打印结果
    │   ├─ success_count = 成功打印的份数
    │   └─ 生成详细报告
    │
    └─ 5. 返回结果
        {
          "success": true,
          "message": "标签打印完成：1/1 成功",
          "printer": "TSC TTP-244",
          "copies": 1,
          "successful": 1,
          "details": [
            {"copy": 1, "result": {"success": true, ...}}
          ]
        }
```

```python
PrinterUtils.print_file(file_path, printer_name)
    │
    ├─ 1. 验证文件存在
    │   └─ os.path.exists(file_path)
    │
    ├─ 2. 使用 win32print API
    │   ├─ OpenPrinter(printer_name)     # 打开打印机
    │   ├─ CreateDC(printer_name)        # 创建设备上下文
    │   ├─ StartDoc()                    # 开始文档
    │   ├─ StartPage()                   # 开始页
    │   ├─ 发送打印数据
    │   │   ├─ 如果是图片 → StretchDIBits()
    │   │   └─ 如果是文档 → TextOut()
    │   ├─ EndPage()                     # 结束页
```

## 错误处理策略

| 错误场景 | 处理策略 |
|---------|---------|
| **单位不存在** | 返回建议列表 (模糊匹配 top 5) |
| **解析失败** | 返回错误码 `NUMBER_MODE_STRICT_FAILED`，不启用预览兜底 |
| **增删改请求但未识别单位** | 提示用户明确单位，例如："七彩乐园再加 1 桶 9803 规格 28" |
| **文档生成失败** | 返回详细错误信息 |
| **未找到标签打印机** | 返回错误提示，建议用户连接或选择打印机 |
| **打印机离线** | 返回 `"status": "离线"`，建议检查连接 |
| **标签生成失败** | 记录日志，继续生成其他标签 |
| **打印失败** | 返回详细错误信息，支持重试 |
| **文档/标签打印机相同** | 验证失败，提示用户配置不同的打印机 |
| **未确认直接打印** | 返回 `print_confirm_required`，前端必须先询问用户 |
| **用户取消打印** | 返回 `print_cancelled`，任务状态置为已取消打印 |

- 标签布局：4 个区域（编号、名称、日期、规格）
