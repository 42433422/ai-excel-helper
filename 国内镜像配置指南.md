# Docker 国内镜像加速配置指南

> **适用于 XCAGI v4.0 AI 员工**

## 🐌 问题解决：无法拉取 Docker 镜像

### 问题原因
由于网络限制，中国大陆地区无法直接访问 Docker Hub (`registry-1.docker.io`)，导致拉取镜像失败。

### 错误信息
```
failed to resolve reference "docker.io/library/redis:7-alpine"
dial tcp 98.159.108.71:443: connectex: A connection attempt failed
```

---

## ✅ 解决方案（3 种）

### 方案 1：使用镜像加速脚本（最简单）⭐

```cmd
双击运行 setup-mirror.bat
```

然后按照提示重启 Docker Desktop。

### 方案 2：手动拉取镜像

```cmd
# 拉取 Redis 镜像
docker pull registry.cn-hangzhou.aliyuncs.com/library/redis:7-alpine

# 拉取 Python 镜像
docker pull registry.cn-hangzhou.aliyuncs.com/library/python:3.11-slim

# 然后运行部署
docker-compose up -d --build
```

### 方案 3：配置 Docker 镜像加速器

#### Windows Docker Desktop

1. 打开 Docker Desktop
2. 点击右上角 ⚙️ Settings
3. 选择 Docker Engine
4. 添加以下配置：

```json
{
  "registry-mirrors": [
    "https://registry.docker-cn.com",
    "https://mirror.ccs.tencentyun.com",
    "https://hub-mirror.c.163.com",
    "https://docker.mirrors.ustc.edu.cn"
  ],
  "debug": false
}
```

5. 点击 "Apply & restart"

#### Linux Docker

编辑 `/etc/docker/daemon.json`：

```json
{
  "registry-mirrors": [
    "https://registry.docker-cn.com",
    "https://mirror.ccs.tencentyun.com",
    "https://hub-mirror.c.163.com"
  ]
}
```

重启 Docker：
```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

## ❓ 常见问题

### Q1: 镜像源也连不上怎么办？

**解决**：
1. 尝试多个镜像源
2. 检查防火墙设置
3. 使用手机热点测试
4. 等待网络恢复

### Q2: 拉取速度很慢怎么办？

**解决**：
- 选择离你地理位置近的镜像源
- 避开网络高峰期
- 使用镜像加速脚本自动配置

### Q3: 配置了镜像源还是失败？

**解决**：
```bash
# 1. 验证配置是否生效
docker info | grep -A 5 "Registry Mirrors"

# 2. 测试镜像源连通性
ping registry.docker-cn.com

# 3. 清除 Docker 缓存
docker system prune -a

# 4. 重启 Docker 服务
```

### Q4: 如何修改 docker-compose.yml 使用国内镜像？

在 `docker-compose.yml` 中指定镜像源：

```yaml
services:
  redis:
    image: registry.cn-hangzhou.aliyuncs.com/library/redis:7-alpine
    # 其他配置...
```
