## 测试环境准备

### 0. 端口说明

| 环境 | 服务 | 端口 | 访问地址 |
|------|------|------|----------|
| **开发环境** | 前端（Vite） | 5001 | http://localhost:5001 |
| **开发环境** | 后端（Flask） | 5000 | http://localhost:5000 |
| **生产环境** | 前端（Nginx） | 80 | http://localhost:80 |
| **生产环境** | 后端（Flask） | 5000 | http://localhost:5000 |
| **生产环境** | Redis | 6379 | localhost:6379 |

**注意**：
- 开发环境使用 `start-dev.bat` 启动，前端运行在 **5001** 端口
- 生产环境使用 Docker 部署，前端通过 Nginx 运行在 **80** 端口
- 本文档中的测试步骤默认使用开发环境端口（5001）

### 1. 环境检查清单

#### 开发环境（本地运行）
```bash
# 1. 检查后端服务（Flask）
curl http://localhost:5000/api/health

# 2. 检查前端服务（Vite 开发服务器）
curl http://localhost:5001/index.html

# 3. 检查数据库连接
curl http://localhost:5000/api/health/db

# 4. 检查 Redis 连接
curl http://localhost:5000/api/health/redis
```

#### 生产环境（Docker 部署）
```bash
# 1. 检查整体服务（Nginx 80 端口）
curl http://localhost:80

# 2. 检查后端 API
curl http://localhost:5000/api/health

# 3. 检查数据库连接
curl http://localhost:5000/api/health/db
```

### 4. 测试前置条件

**数据库初始化**：
```bash
# 1. 确保数据库已创建

# 2. 运行数据库迁移（如需要）
alembic upgrade head

# 3. 初始化测试数据（如需要）
python scripts/init_test_data.py
```

**微信集成前置条件**：
```bash
# 1. 确保微信解密模块已安装
ls -la resources/wechat-decrypt/

# 2. 确保微信数据库可访问
```

## 常见问题排查

### 1. 服务无法访问

**问题**：后端服务无响应

```bash
# 检查服务状态
docker-compose ps
systemctl status xcagi-backend

# 检查端口占用
netstat -ano | findstr :5000

# 查看错误日志
docker-compose logs backend
```

**解决方案**：
1. 重启服务：`docker-compose restart`
2. 检查端口占用并杀掉进程
3. 检查防火墙设置

### 2. 前端无法加载

**问题**：访问 http://localhost:5001 白屏

```bash
# 检查 Node.js 进程
tasklist | findstr node

# 检查前端构建产物
ls -la templates/vue-dist/

# 重新构建前端
cd frontend && npm run build
```

**解决方案**：
1. 确保后端已启动并运行在 5000 端口
2. 检查前端代理配置
3. 清除浏览器缓存

### 3. AI 功能无响应

**问题**：AI 聊天超时或无响应

```bash
# 检查 AI 模型状态
curl http://localhost:5000/api/ai/model-status

# 检查 GPU 驱动（如使用 GPU）
nvidia-smi

# 检查模型文件
ls -la models/
```

**解决方案**：
1. 确认 AI 模型已加载
2. 检查 GPU/CUDA 是否正常
3. 查看 AI 服务日志

### 4. 数据库连接失败

**问题**：数据库连接错误

```bash
# 检查数据库文件
ls -la data/xcagi.db

# 检查数据库权限
chmod 755 data/

# 重置数据库（如需要）
python scripts/reset_db.py
```

**解决方案**：
1. 确保数据库文件存在
2. 检查文件权限
3. 重新初始化数据库

### 5. 微信集成失败

**问题**：微信消息扫描失败

```bash
# 检查微信解密模块
ls -la resources/wechat-decrypt/

# 检查微信数据库
ls -la resources/decrypted/message/

# 测试微信插件
curl http://localhost:5000/api/wechat/plugin-status
```

**解决方案**：
1. 确保微信已登录并解密
2. 检查数据库路径配置
3. 重新运行微信解密工具

### 6. 打印功能异常

**问题**：无法连接到打印机

```bash
# 检查打印机服务
curl http://localhost:5000/api/print/printers

# 检查 Bartender 服务
tasklist | findstr bartender

# 检查网络打印机
ping printer_ip
```

**解决方案**：
1. 确保打印机已开机并连接
2. 检查打印机驱动安装
3. 确认网络打印机可访问

## 测试注意事项

### 环境相关
- 测试前确认所有服务正常运行
- 测试数据与生产数据严格隔离
- 测试完成后清理测试数据

### 权限相关
- 测试不同角色账号的功能
- 验证权限边界
- 测试未登录访问被拒绝

### 数据相关
- 测试数据使用测试账号
- 不要使用真实的客户数据
- 测试完成后删除测试数据

### 安全相关
- 测试时不要输入真实敏感信息
- 测试 SQL 注入等攻击时注意数据安全
- 记录安全漏洞但不要利用

### 性能相关
- 性能测试需在隔离环境进行
- 记录性能指标作为基准
- 发现性能问题时记录当时负载

### 回归测试
- Bug 修复后必须进行回归测试
- 优先测试相关模块
- 记录回归测试结果
