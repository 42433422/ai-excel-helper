# 数据库管理系统使用说明

## 🎯 系统概述

这是一个专为发货单生成系统设计的数据库管理界面，让您能够以客户单位为中心查看和管理产品信息。

## 📋 功能特性

### ✅ 主要功能
- **客户单位管理**: 查看所有客户单位信息
- **产品列表查看**: 根据选择的客户显示对应产品
- **产品搜索**: 实时搜索产品名称或型号
- **产品管理**: 添加、编辑、删除产品信息
- **数据统计**: 实时显示产品数量统计

### ✅ 界面特点
- **响应式设计**: 适配各种屏幕尺寸
- **美观界面**: 使用Tailwind CSS设计
- **实时搜索**: 即时筛选产品结果
- **模态框操作**: 优雅的弹窗操作界面

## 🚀 使用方法

### 1. 启动系统
```bash
# 在AI助手目录下启动后端服务
python app_api.py
```

### 2. 访问界面
在浏览器中打开：`database_management.html`

### 3. 操作流程

#### 查看客户产品
1. 从"选择客户单位"下拉菜单中选择客户
2. 系统自动加载该客户的所有产品
3. 产品信息包括：型号、名称、规格、价格、单位、状态

#### 搜索产品
1. 在"搜索产品"框中输入关键词
2. 系统实时筛选匹配的产品

#### 添加新产品
1. 点击"添加产品"按钮
2. 填写产品信息（型号、名称、价格等）
3. 点击"添加产品"确认

#### 编辑产品
1. 在产品表格中点击编辑按钮（✏️）
2. 修改相关信息

#### 删除产品
1. 在产品表格中点击删除按钮（🗑️）
2. 确认删除操作

## 📊 API接口

### 客户管理
- `GET /api/customers` - 获取所有客户单位
- `GET /api/products/{customer_id}` - 获取指定客户的产品

### 产品管理
- `POST /api/products` - 添加新产品
- `PUT /api/products/{id}` - 更新产品信息
- `DELETE /api/products/{id}` - 删除产品

## 🗂️ 数据库结构

### 主要表结构
- **purchase_units**: 采购单位（客户）
- **products**: 产品信息
- **customer_products**: 客户产品关联表

## ⚠️ 注意事项

1. **数据备份**: 修改产品信息前请确保数据库已备份
2. **价格管理**: 产品价格直接影响发货单计算
3. **客户关联**: 产品与客户的关联关系需要正确维护
4. **数据验证**: 系统会自动验证必填字段

## 🔧 技术架构

- **前端**: HTML5 + JavaScript + Tailwind CSS
- **后端**: Python Flask + SQLite
- **API**: RESTful API设计
- **数据库**: SQLite (products.db)

## 📞 技术支持

如有问题，请检查：
1. 后端服务是否正常运行
2. 数据库文件是否存在
3. 浏览器控制台是否有错误信息
