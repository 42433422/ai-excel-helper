## 🔧 打印机问题诊断

### 问题症状
- ❌ 两个任务都发送到同一台打印机
- ❌ 发货单和标签打印机混乱
- ❌ 默认打印机干扰

### 诊断工具
**1. API诊断接口**
- 访问: http://127.0.0.1:5000/api/print/diagnose
- 功能: 自动检测打印机配置和冲突

**2. 诊断脚本**
```cmd
cd "C:\Users\Administrator\Desktop\新建文件夹 (4)\AI助手"
python printer_diagnostic.py
```
- 功能: 详细的打印机配置分析

### 打印机诊断输出示例
```
=== 系统打印机列表 ===
1. Microsoft Print to PDF (类型: 通用打印机)
2. TSC TTP-244 Plus (类型: 条码/标签打印机)
3. Jolimark 24-pin printer (类型: 点阵/发货单打印机)

=== 打印机分离性验证 ===
✅ 打印机配置验证通过
📄 发货单打印机: Jolimark 24-pin printer
🏷️ 标签打印机: TSC TTP-244 Plus
```

### 解决方案

**问题1: 只有一个打印机**
- **解决方案**: 安装独立的标签打印机或设置手动打印模式

**问题2: 打印机名称相同**
- **解决方案**: 
  1. 重新命名打印机
  2. 重新安装驱动程序
  3. 检查Windows打印机设置

**问题3: 未检测到打印机**
- **解决方案**: 
  1. 检查物理连接
  2. 验证驱动程序
  3. 测试Windows打印功能

## ⚠️ 故障排除

**如果启动器闪退：**
1. 右键点击 → 以管理员身份运行
2. 使用 `稳定启动器.bat`（包含完整错误处理）
3. 或使用命令行方式启动

**如果自动打开页面失败：**
1. 确认系统已启动（等3-5秒）
2. 手动访问: http://127.0.0.1:5000
3. 检查端口5000是否被占用

**如果打印功能异常：**
1. 检查打印机是否连接和开机
2. 确认打印机名称是否正确
3. 查看Flask日志中的打印结果
4. 使用打印机诊断工具检测冲突

**如果打印机冲突：**
1. 运行 `python printer_diagnostic.py` 进行诊断
2. 检查API诊断接口: `/api/print/diagnose`
3. 确保两台打印机名称不同
4. 重新安装或重命名打印机

**如果数据库页面无法访问：**
1. 确认系统已启动
2. 检查数据库文件是否存在
3. 查看Flask日志是否有错误信息
