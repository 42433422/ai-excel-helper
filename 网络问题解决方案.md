# Docker 网络连接问题解决方案

## ❌ 错误信息

```
failed to resolve reference "docker.io/library/redis:7-alpine"
dial tcp 98.159.108.71:443: connectex: 
A connection attempt failed because the connected party did not properly 
respond after a period of time, or established connection failed because 
connected host has failed to respond.
```

## 🔍 问题原因

由于网络限制，中国大陆地区无法直接访问 Docker Hub，导致：
- 无法拉取 `redis:7-alpine` 镜像
- 无法拉取 `python:3.11-slim` 镜像
- Docker 部署失败

---

## ✅ 解决方案（4 种）

### 方案 1：配置 Docker 镜像加速器 ⭐⭐⭐ 推荐

#### Windows Docker Desktop

1. **打开 Docker Desktop**
2. **点击右上角 ⚙️ Settings**
3. **选择 Docker Engine**
4. **修改配置文件**，添加镜像加速器：

```json
{
  "registry-mirrors": [
    "https://hub-mirror.c.163.com",
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ],
  "debug": false
}
```

5. **点击 "Apply & restart"**

#### 验证配置

```bash
# 查看配置是否生效
docker info | grep -A 5 "Registry Mirrors"
```

---

### 方案 2：使用手机热点 ⭐⭐ 快速临时方案

1. **用手机开启热点**
2. **电脑连接手机热点**
3. **拉取镜像**：

```cmd
cd E:\FHD\XCAGI
docker pull redis:7-alpine
docker pull python:3.11-slim
```

4. **拉取完成后**切回原网络
5. **运行部署脚本**：

```cmd
双击 deploy.bat
```

---

### 方案 3：离线导入镜像 ⭐⭐ 适合团队

#### 步骤 1：在有网络的电脑上下载

```bash
# 下载镜像为 tar 文件
docker save -o redis_7_alpine.tar redis:7-alpine
docker save -o python_3_11_slim.tar python:3.11-slim
```

#### 步骤 2：拷贝到本机

将 `.tar` 文件复制到目标电脑

#### 步骤 3：导入镜像

```cmd
# 加载镜像
docker load -i redis_7_alpine.tar
docker load -i python_3_11_slim.tar
```

#### 步骤 4：验证

```bash
# 查看本地镜像
docker images
```

#### 步骤 5：部署

```cmd
双击 deploy.bat
```

---

### 方案 4：修改 docker-compose.yml 使用国内镜像源

编辑 `docker-compose.yml`，将 Redis 镜像改为国内源：

```yaml
services:
  redis:
    # 原配置
    # image: redis:7-alpine
    
    # 修改为（选择一个可用的）
    image: dockerproxy.cn/library/redis:7-alpine
    # 或
    image: swr.cn-east-3.myhuaweicloud.com/library/redis:7-alpine
```

## 📋 完整部署流程（国内环境）

### 第一步：配置镜像加速

```cmd
# 方法 A: 手动配置
1. 打开 Docker Desktop Settings
2. Docker Engine
3. 添加 registry-mirrors 配置
4. Apply & restart

# 方法 B: 运行配置脚本
双击 setup-mirror.bat
然后手动重启 Docker Desktop
```

### 第二步：测试镜像拉取

```cmd
# 测试拉取 Redis
docker pull redis:7-alpine

# 测试拉取 Python
docker pull python:3.11-slim
```

如果成功，继续下一步。如果失败，尝试其他镜像源。

### 第三步：运行部署

```cmd
双击 deploy.bat
选择：1 (生产环境部署)
```

### 第四步：验证部署

```cmd
# 查看容器状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 访问应用
http://localhost
```

## 🔍 故障排查

### 检查点 1：Docker 是否正常运行

```bash
# 查看 Docker 信息
docker info

# 查看 Docker 版本
docker --version
```

### 检查点 2：网络连接

```bash
# 测试 DNS 解析
ping docker.io

# 测试镜像源连通性
ping hub-mirror.c.163.com
```

### 检查点 3：镜像配置

```bash
# 查看当前镜像源配置
docker info | grep -A 5 "Registry Mirrors"

# 查看 daemon.json 配置
type C:\ProgramData\docker\config\daemon.json
```

### 检查点 4：清除缓存

```bash
# 清除未使用的镜像
docker image prune -a

# 清除构建缓存
docker builder prune -a
```
